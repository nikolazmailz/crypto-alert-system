# [ADR-010] Использование Resilience4j для обеспечения отказоустойчивости (Circuit Breaker & Bulkhead)

**Status:** Accepted
**Date:** 2026-03-26

## Context
Наш сервис market-data-service зависит от внешнего API (Binance).
В распределенных системах внешние зависимости могут быть нестабильными (задержки, сетевые сбои, плановые работы).
Без механизмов защиты задержка в Binance может привести к исчерпанию ресурсов (потоков) нашего приложения,
вызывая каскадное падение всей системы.

## Decision
Для защиты системы мы внедряем библиотеку Resilience4j со следующими паттернами:
* Circuit Breaker (Предохранитель): Для предотвращения бесконечных запросов к упавшему сервису. При превышении порога ошибок (failureRateThreshold: 50%) предохранитель размыкается, и запросы блокируются на время восстановления.
* Bulkhead (Перегородка): Для изоляции ресурсов. Мы ограничиваем количество параллельных вызовов к конкретной интеграции (maxConcurrentCalls: 10), чтобы медленные ответы одного провайдера не занимали все ресурсы реактивного рантайма.
* Retry (Повторы): Для обработки кратковременных сетевых «шорохов» (например, 3 попытки с задержкой 500мс).

## Technical Implementation
* Использование функционального подхода через decorateSuspendFunction вместо аннотаций. Это обеспечивает прозрачность цепочки вызовов и лучшую поддержку корутин Kotlin без магии Spring AOP.
* Настройка через application.yml с использованием базовых конфигураций (configs) для удобного масштабирования при добавлении новых провайдеров.

## Consequences

**Положительные:**
* **Fail-fast:** Система мгновенно возвращает ошибку, если ресурс недоступен, не заставляя клиента ждать тайм-аута.
* **Изоляция:** Сбой в модуле рыночных данных не повлияет на работу других будущих модулей (например, portfolio-manager).
* **Наблюдаемость:** Статусы предохранителей можно мониторить через Spring Boot Actuator (registerHealthIndicator=true).

**Отрицательные:**
* **Входной порог:** Дополнительная сложность конфигурации и необходимость обработки исключений CircuitBreakerOpenException и BulkheadFullException.
